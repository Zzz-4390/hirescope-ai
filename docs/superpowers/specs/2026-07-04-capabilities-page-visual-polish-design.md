# 产品能力页精准渐进式精修设计

## 目标

以 `imgs/产品能力页/` 中完整设计稿和 01–08 分区设计稿为最高视觉基准，在不改变页面结构、文案含义、导航和业务逻辑的前提下，提高产品能力页的视觉还原度、SaaS mockup 质感与滚动进入体验。

## 范围

允许修改：

- `apps/web/src/app/capabilities/page.tsx`
- 产品能力页专属组件和测试
- `apps/web/src/app/globals.css` 中明确以 `.cap-*` 或产品能力页父级限定的样式

禁止修改：

- 首页和登录页
- 路由、接口、导航逻辑、业务逻辑和全局状态
- 当前首页相关未提交改动
- 页面 section 顺序和文案含义

## 视觉系统

- 内容容器、左右比例和各 section 间距以分区设计稿为准。
- 保持白色主背景，淡蓝渐变仅用于分区衔接和层次增强。
- mockup 统一使用轻边框、克制圆角、柔和多层阴影和清晰内部分隔。
- 标题采用更清晰的字重和紧凑字距；说明文字、标签和数据建立稳定层级。
- 小功能点统一图标尺寸、描边粗细、基线、文字间距和两列网格节奏。
- 不新增与设计稿无关的装饰、文案、卡片系统或视觉风格。

## 首屏

- 调整左右比例，使右侧六步流程卡更宽，更接近设计稿。
- 六个步骤保持等宽，强化竖向分割线、图标容器、编号和标题对齐。
- 底部使用虚线轨道、节点和回环线；“完整评估闭环”居中。
- 标题控制换行，保证“能力报告”不被拆开。
- 首屏背景使用非常克制的淡蓝层次，不影响文字对比度。

## 01–05 产品 mockup

- 统一产品窗口外框、圆角、边框、阴影、表面颜色和内部间距。
- 01 保持侧栏、项目摘要、技术栈和目录结构的真实后台布局。
- 02 强化文件树、代码区、质量评分和 AI 建议之间的分栏与状态层级。
- 03 强化面试进度、对话消息、输入框、评分进度条和状态标识。
- 04 强化综合评分、雷达图、AI 总结、优缺点和改进建议的报告感。
- 05 强化授权配置、复选项、有效期、密码开关、链接预览和撤销动作。
- 仅补充展示细节，不新增业务交互或状态逻辑。

## 06–07

- 06 使用设计稿风格的人物头像或精致本地 avatar，移除粗糙灰色字母头像；保留候选人摘要、评分、标签和建议层级。
- 07 重做统计卡、趋势折线图和技术栈条形图的视觉细节，加入清晰数据标签、克制配色、柔和网格/面积和真实后台卡片层级。

## Reveal 动画

- 新增产品能力页专属客户端 Reveal 管理组件，使用 `IntersectionObserver`。
- 每个 section 进入视口时从 `opacity: 0`、`translateY(28px)` 过渡到正常状态。
- 时长约 `600ms`，缓动为 `cubic-bezier(0.22, 1, 0.36, 1)`。
- section 内按标题、说明、功能点、右侧 mockup 分层出现；mockup 相对左侧约延迟 `140ms`。
- 首屏在客户端初始化前保持可见，初始化后再启用动画类，避免刷新闪烁。
- 每个 section 仅播放一次，动画不接管滚轮、不改变滚动位置、不启用整屏吸附。

## 无障碍与响应式

- `prefers-reduced-motion: reduce` 时禁用透明度、位移、延迟和过渡，内容直接显示。
- 移动端取消易引发布局抖动的位移和分层延迟，保持自然文档流。
- 保证页面无横向溢出；宽 mockup 在移动端采用现有可控滚动或合理缩放策略。
- 动画只使用 `opacity` 和 `transform`，不触发布局重排。

- 使用应用内浏览器验证桌面端和移动端。
- 检查 01–07 Reveal、首屏刷新、自然滚动、标题换行、06 avatar、07 图表和控制台错误。
- 将实现截图与完整稿及分区稿逐项比较，至少覆盖布局比例、字体层级、mockup、流程闭环、06、07、移动端和动画。
- 复核 Git diff，确认未触碰首页、登录页、路由、接口、导航逻辑和业务逻辑。
