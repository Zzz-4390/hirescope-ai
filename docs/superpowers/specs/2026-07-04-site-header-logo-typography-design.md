# SiteHeader Logo 与导航字重修复设计

## 目标

在保持全站唯一 `SiteHeader` 视觉契约的前提下，对照参考截图和 `imgs/logo.png` 恢复轻量 SaaS 导航质感，并将当前三条粗矩形 Logo 改为精致、可缩放的多段斜向短条 SVG。

## Logo 图形

- 继续使用代码原生 SVG，不使用低清位图。
- 使用 SVG 坐标/path 直接绘制多段长度不同的圆角短条，组成错落节奏和约 `-45deg` 的整体方向；不通过复用大矩形后整体 rotate 实现。
- 各短条保持一致的基础粗细，但通过长度、起点和蓝色层次形成原设计稿的连续感。
- SVG 使用稳定 `viewBox`、清晰圆角和可缩放路径/矩形，不再使用 3 个大号旋转矩形。
- 图形与品牌文字垂直居中，桌面端保持 42px 视觉框，移动端按现有断点缩小。

## 品牌文字

- 中文“码途 AI”：约 `18px / 650`，紧凑但不发黑。
- 英文“HireScope AI”：约 `9px / 400`，降低对比度并保持清晰。
- 调整上下间距、行高和字距，使文字块与图标光学居中。

## 导航字重

- 主导航与“登录”：`15px`；若当前字体不稳定支持 `450`，统一使用 `400`。
- “立即体验”：蓝色，使用约 `500` 的中等字重，不使用粗体按钮感。
- active 仅改变下划线；除 active 位置外，不改变字体尺寸、字重、间距或布局。

## 范围

- 修改 `Logo.tsx` 的 SVG 几何。
- 修改 `globals.css` 中唯一全局 SiteHeader/Logo/导航规则。
- 增加 Logo 结构与全局样式契约测试；测试验证多段结构并禁止旧三矩形方案，但不锁死每个 SVG 坐标。
- 不新增页面级 header 覆盖，不修改 `SiteHeader` 导航逻辑、页面内容、路由、接口或业务逻辑。

## 验证

- 先运行失败测试，证明旧 Logo 仍是 3 个大矩形且导航字重偏高。
- 运行 web tests、typecheck、lint 和 build。
- 在相同桌面视口分别截图首页、产品能力页和登录页。
- 比较 header、Logo、nav、actions 边界框与计算样式；除 active 项和下划线外完全一致。
- 对照参考图检查斜向短条节奏、品牌文字层级和导航轻量感。
